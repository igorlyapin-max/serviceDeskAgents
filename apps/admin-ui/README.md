# Admin UI

Административная консоль AI ServiceDesk Platform.

Локальный URL при запущенном orchestrator:

```bash
http://127.0.0.1:18088/admin
```

MVP обслуживается FastAPI как static UI и использует только Admin API. Изменения конфигурации выполняются через процесс `draft -> validate -> regression -> activate`; опасные действия проходят через RBAC и audit.

Раздел `Сценарии обработки -> 1. Разрешение атрибутов` управляет профилями заполнения слотов как resolver: признаки, операция разрешения, правила оценки результата операции, матрица решений, уточняющий вопрос и пакет передачи человеку настраиваются формовыми полями без JSON-редактора.

Раздел `Сценарии обработки -> Каналы взаимодействия` задает режим работы сценария: мессенджер-бот, сервисдеск или отладочный режим. Канал определяет, куда задаются вопросы, сколько ждать ответа, что делать с незавершенным обсуждением и какое действие выполнять при эскалации.

Блок `3. ReAct-планирование` задает рамки планирования: максимум итераций, число ошибок ReAct-вызовов подряд до передачи, разрешенные группы действий ReAct и стоп-условия. Группы действий и стоп-условия выбираются чекбоксами с пояснениями.

Блок `4. ReAct-вызовы и матрица запуска` показывает один `Вид запуска` для каждого ReAct-вызова ИИ. Карточка запуска разделена на группы: `ReAct-вызов ИИ`, `Текущая привязка операции`, `Параметры вызова` и `Контроль запуска`. В матрице выбирается только ReAct-вызов как бизнес-действие агента; endpoint и операция подтягиваются автоматически из раздела `Привязка операций`. В `Параметры вызова` параметры берутся из схемы вызова, а правило заполнения выбирается как `slot`, `case`, `context`, `constant` или `secret`. Необязательные параметры свернуты отдельно. `required_slots` не редактируется вручную: UI вычисляет его из всех `parameter_bindings` вида `slot:<slot_id>`. Риск, аудит, логирование и согласование относятся к сценарной политике запуска.

Исполнение разделено на три справочника. `Интеграции` управляет техническими подключениями и операциями внутри них, сгруппированными по типу адаптера (`mock`, `n8n webhook`, `direct http`, `queue`). Операция содержит `request_schema` для входного payload и тестовый ответ mock. `ReAct-вызовы ИИ` хранит бизнес-действия, которые может выбрать модель в ReAct-loop, их схемы параметров, схемы результата и политику риска. Ограничения по окружениям в ReAct-вызовах не настраиваются, пока нет надежного источника этой информации. `Привязка операций` связывает один ReAct-вызов ИИ с одной текущей операцией подключения и хранит маппинг `параметр операции -> параметр вызова / константа / секрет`; новая привязка заменяет старую. `Отвязать` доступно только если ReAct-вызов не используется в матрицах запуска, профилях разрешения атрибутов или каналах взаимодействия.

Фактические действия эскалации настраиваются профилями действий канала. Блок `5. Решение и эскалация` задает условия решения, например порог Major Incident, условия передачи и пакет передачи. Условия и пакет выбираются чекбоксами; обязательные элементы пакета нельзя отключить. Runtime выбирает профиль действия автоматически по `event_type` выбранного канала сценария.

Раздел `Модели` управляет подключениями LiteLLM:

- `vLLM CPU` для локальной разработки;
- `OpenAI API` для реальной модели через token из переменной окружения.
- дополнительные подключения LiteLLM, которые создаются, редактируются и удаляются из UI.

Token не отображается в UI. В интерфейсе указывается имя переменной окружения, например `OPENAI_API_KEY`; значение можно ввести в поле `Значение секрета`. После сохранения поле очищается, а интерфейс показывает только `параметр скрыт` или `параметр не заполнен`.

Для нового подключения без отдельной записи в `infra/litellm/litellm.yaml` в поле `Alias в LiteLLM` используйте provider-prefixed model, например `openai/gpt-4.1-mini`. Если нужен собственный короткий alias, он должен быть объявлен в LiteLLM.

Для MVP единое место ключа - файл `.env` в корне репозитория. Файл уже исключен из git через `.gitignore`; скопируйте значения из `.env.example` и заполните:

```bash
MODEL_ACTIVE_PROVIDER=openai
OPENAI_API_KEY=sk-...
OPENAI_MODEL=openai/gpt-4.1-mini
```

Docker Compose читает этот файл автоматически, а локальный orchestrator подхватывает его при импорте приложения. Placeholder и пустой `OPENAI_API_KEY` считаются не настроенным ключом.

При вводе секрета через UI значение сохраняется в локальный `.env` и применяется в текущем процессе orchestrator. Для уже запущенного контейнера LiteLLM может потребоваться перезапуск, чтобы он перечитал переменные окружения.

Профили являются стабильными настройками: можно изменить параметры каждого подключения отдельно, сохранить их, а затем переключаться кнопкой `Сделать активным`. Переключение меняет активное подключение и маршруты по умолчанию, но не стирает настройки остальных подключений. Базовые профили `vLLM CPU` и `OpenAI API` не удаляются из UI, чтобы оставался bootstrap-путь; их можно отключить.
