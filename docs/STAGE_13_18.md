# Этап 13.18. Разделение параметров вызова и операции endpoint

## Назначение

Этап разделяет два уровня параметров:

- параметры вызова: смысловые данные, которые собирает сценарий;
- параметры операции endpoint: технический payload, который получает adapter.

## Изменения

- В операции подключения добавлен `request_schema`.
- В `tool.endpoint_bindings` добавлен `parameter_mapping`.
- Runtime собирает `operation_parameters` перед dispatch и валидирует их по `operation.request_schema`.
- В Admin UI блок `4` переименован в `Параметры вызова`; технический payload операции редактируется только в `Привязка операций`.
- В Operator UI шаг 4 показывает параметры вызова отдельно от подключения и операции.

## Правило настройки

Сценарий заполняет только параметры вызова. Преобразование этих параметров в payload endpoint выполняется в привязке операции.
