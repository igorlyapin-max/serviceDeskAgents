# Этап 13.5: каналы взаимодействия

Этап 13.5 убирает из сценариев жесткую привязку к уровням Л1/Л2. Вместо этого сценарий выбирает канал взаимодействия, а канал определяет, куда задавать вопросы, сколько ждать ответ, что делать с незавершенным обсуждением и как выполнять эскалацию.

## Реализовано

- Новый домен Config Registry `interaction_channels`.
- Новый контракт `contracts/config/interaction-channels.schema.json`.
- Default-каналы:
  - `messenger_bot`: онлайн-интерактивный диалог с пользователем, короткие таймауты, эскалация через действие "позвать специалиста";
  - `service_desk`: офлайн-интерактивная работа через заявку, ожидание по SLA, эскалация через создание наряда;
  - `debug`: локальный режим MVP, вопросы показывает операторская панель, эскалация останавливает сценарий с сообщением.
- Сценарий получил связь:
  - `default_channel_id`;
  - `allowed_channel_ids`.
- Admin UI:
  - раздел `Сценарии обработки -> Каналы взаимодействия`;
  - операции создать, модифицировать, удалить;
  - настройка доставки вопроса, ожидания, действия при отсутствии ответа, незавершенного обсуждения и эскалации;
  - запрет удаления канала, если он используется сценарием.
- Operator UI:
  - показывает канал сценария;
  - показывает способ доставки вопроса;
  - показывает таймауты и действие эскалации в шаге `5. Решение и эскалация`.
- Этапы 13.6-13.7 расширяют эту модель профилями действий канала: блок `5` задает условия решения, а профиль действия выбирается автоматически по `event_type` текущего канала.
- Маршруты классификации переименованы в нейтральные значения:
  - `agent_with_confirmation`;
  - `human_review`;
  - `major_incident`.

## Архитектурное решение

Уровни поддержки не должны быть базовыми сущностями оркестратора. Для MVP есть три режима исполнения:

- мессенджер-бот: вопрос уходит пользователю, эскалация зовет специалиста в диалог;
- сервисдеск: вопрос фиксируется в заявке, эскалация создает наряд;
- отладка: вопрос и остановка остаются в операторском интерфейсе.

Сценарий выбирает готовый канал, но не описывает внутри себя таймеры и действия канала. Это оставляет место для будущего автоисполнения без переписывания сценарной модели.
