# Этап 13.6: профили действий канала

Этап 13.6 уточняет границу между блоком `5. Решение и эскалация` и `Каналами взаимодействия`.

Блок `5` решает, когда нужна передача, ожидание или Major Incident. Канал взаимодействия решает, как выполнить это действие: показать сообщение в отладке, создать наряд, позвать специалиста или оповестить дежурных.

## Реализовано

- В `interaction_channels` добавлены `action_profiles`.
- В профиль действия канала входят:
  - название;
  - тип события;
  - действие канала;
  - optional tool;
  - подключение;
  - операция;
  - шаблон сообщения.
- Добавлено действие `notify_on_call`.
- В Admin UI блока `5. Решение и эскалация` убран переключатель `Оповещать дежурных`.
- В Admin UI `Каналы взаимодействия` добавлены карточки профилей действий канала.
- Operator UI показывает вычисленные профили канала и их действия.
- Dry-run сценария возвращает `channel_action_profiles`.

Этап 13.7 дополнительно убирает из блока `5` ручной выбор профилей канала. Профили вычисляются автоматически по `event_type` текущего канала сценария.

## Default-профили

- `messenger_bot`:
  - обычная передача: позвать специалиста в чат;
  - нет ответа: создать черновик;
  - Major Incident: оповестить дежурных;
  - policy blocked: позвать специалиста.
- `service_desk`:
  - обычная передача: создать наряд;
  - нет ответа: создать наряд;
  - Major Incident: создать срочный наряд дежурной группе;
  - policy blocked: создать наряд для ручной проверки.
- `debug`:
  - все профили останавливают сценарий и показывают сообщение оператору.
