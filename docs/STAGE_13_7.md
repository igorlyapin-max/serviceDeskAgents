# Этап 13.7: автоматический выбор профилей канала

Этап 13.7 упрощает связь блока `5. Решение и эскалация` с каналами взаимодействия.

Сценарий уже выбирает канал. Канал уже содержит профили действий. Поэтому блок `5` больше не выбирает профили канала вручную. Runtime определяет нужный профиль автоматически:

```text
scenario.default_channel_id -> interaction_channel.action_profiles[event_type]
```

## Реализовано

- Из `escalation_policies` убран пользовательский `channel_profile_mapping`.
- Из Admin UI блока `5. Решение и эскалация` убран блок `Профили канала`.
- В блоке `5` остались только условия решения:
  - автозакрытие;
  - ожидание;
  - условия передачи;
  - Major Incident threshold;
  - пакет передачи;
  - шаблон уведомления пользователя.
- В `Каналах взаимодействия` профили действий остались единственным местом настройки фактической доставки.
- Backend нормализует старые payload: удаляет `channel_profile_mapping` и legacy `major_incident.notify_on_call`.
- Dry-run продолжает возвращать `channel_action_profiles`, но теперь это вычисляемое представление канала, а не результат выбора в блоке `5`.

## Валидация

- В каждом канале должен быть ровно один профиль для каждого обязательного `event_type`:
  - `standard_handoff`;
  - `no_answer`;
  - `major_incident`;
  - `policy_blocked`.
- Дубликаты обязательных `event_type` запрещены.
- Сценарий невалиден, если его канал не содержит полный набор обязательных профилей.
