# Локальный источник базы знаний

Здесь лежат документы для начального RAG-контура разработки.

Эта директория является только одним входом коннектора источников, а не всей архитектурой базы знаний. Корпоративная wiki, ServiceDesk KB, выгрузки CMDB, Git-репозитории и будущие API-источники должны описываться в каталоге источников и нормализоваться при ingestion перед использованием в retrieval.

Начальные типы материалов:

- FAQ.
- Ранбуки.
- Архитектурные заметки.
- Документация сервисов.
- Выгрузки CMDB.

Перестроение базы знаний является действием оператора. Анализ заявки должен использовать последний успешно построенный индекс и не должен синхронно перестраивать индекс или загружать удаленные источники.
